📌 Kafka UI Setup & Usage Guide

Steps to install Docker, create a reusable Kafka UI container, and restart it anytime.

🧩 1. Install Docker (if not installed)

sudo apt update
sudo apt install docker.io -y
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker $USER
# logout & login again after this step

📦 2. Create Kafka UI Container (run once)

docker create --name kafka-ui \
  -p 8080:8080 \
  -e KAFKA_CLUSTERS_0_NAME=prod \
  -e KAFKA_CLUSTERS_0_BOOTSTRAPSERVERS=52.172.216.129:9095 \
  provectuslabs/kafka-ui

🚀 3. Start Kafka UI

docker start kafka-ui

🛑 4. Stop Kafka UI

docker stop kafka-ui

🔍 5. Check Container Status

docker ps -a

🌐 6. Access Kafka UI

http://localhost:8080

🧹 7. Remove old/unused container (optional)

docker rm kind_jackson

📖 Notes

  • Kafka UI connects using the broker you provided: 52.172.216.129:9095
  • Telepresence + VPN must be active for cluster access.
  • The container named kafka-ui persists config — no need to recreate it.
  • Open Messages tab inside Kafka UI to inspect live topic data.

⚡ Setup Instructions - Follow These Steps First!

  1. Connect to VPN: Ensure you're connected to your organization's VPN before proceeding.
  2. Connect via Telepresence: Try connecting using these commands sequentially:
    • telepresence connect (try this first)
    • telePresenceNew connect (if the first one fails)
    Note: Two different telepresence binaries are listed as backup options for connection stability.
  3. Navigate to Kafka Directory: Open terminal and run: cd ~/kafka_2.13-3.9.1/
  4. Ready to Go: Now you can copy and run commands from the dashboard below!
